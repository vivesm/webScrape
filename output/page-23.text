This guide describes how to run the embedded Identity Engine SDK or Sign-In Widget sample app. Make sure to successfully 
create and set up your Okta org
 first. You also need to 
download and set up the SDK
.

The embedded SDK flow 
The SDK integrates into your app by using a native language interface and communicates directly with the Okta APIs by using a REST interface. The following diagram illustrates the SDK flow.
Run the embedded SDK sample app 
If you haven't already done so, set up your Okta org by completing these steps:
Set up your Okta org for a password factor only use case
Download and set up the sample app
Using Android Studio, open the 
okta-idx-android
 project.
Open the 
okta.properties
 file in the project root directory and 
configure it
 with the settings from your org.
Click 
Run
 to launch the application in the Android emulator.
Tap 
LOGIN
 to start the authentication flow.
Work with the use cases 
After you successfully run the sample app, you can build your own integration by using the sample app as your guide. Explore use cases that are available with the SDK, starting with the 
Basic sign-in flow example with the password factor
 use case.
